# Changelog (miniY)

음악 봇 프로젝트 `miniY` 의 모든 업데이트 내역(새 기능, 버그 수정 등)이 이 파일에 문서화됩니다.

---

## [1.2.0] - 2026-02-26
### ✨ 기능 추가 (Added)
- **대기열 관리 고도화**: 대기열에 곡을 무작위로 섞을 수 있는 `[🔀 셔플]` 버튼 UI 추가.
- **클릭 기반 곡 제명 기능**: 대기열 뷰에서 `[🗑️ 삭제]` 버튼을 통해 특정 인덱스를 지정하여 대기열에서 뺄 수 있는 `QueueDeleteView`를 신규 도입.
- **슬래시 명령어 별칭 기능**: 디스코드 채팅창에서 `/ㅇ`, `/노래`, `/재생` 등의 한글 기반 매핑 별칭(Aliases)을 지원하여 접근성 대폭 상향.

### 🐛 버그 수정 (Fixed)
- **타입 에러 수정(진행바 폭사 해결)**: `music.py` 내부 진행 시간(`05:22` 등)을 문자열로 다듬어주는 `format_duration` 헬퍼 함수가 파이썬 실수형(`float`) 데이터를 처리하려다가 `update_controller` 배경 작업 루프를 통째로 멈춰버리는 고질적인 타입 에러를 고쳤습니다.
- **유튜브 자막(HTTP 429) 블락 우회 성공**: 디스코드 서버가 봇 IP망 기반으로 영상을 크롤링할 때 `yt-dlp` 및 기본 HTTP 통신망에서 "Too Many Requests" 차단을 받던 증상을 완벽하게 피할 수 있도록 파싱 엔진을 `youtube-transcript-api` 로 통째로 교체. (한국어, 영어, 일본어, 자동 생성 자막 완벽 지원)

### 📈 개선 (Changed)
- 라이브 진행 컨트롤러 메시지와 정적인 재생 정보(Now Playing) 엠베드를 2개로 분리하여 시인성을 높이고 API 부하 감소.
- 빈 여백이나 음악 사이사이에 텍스트가 깜빡이는 것을 막기 위해 이전 자막 텍스트를 다음 자막이 나올 때까지 화면에 **계속 홀드시키는 유지 알고리즘** 구현.

## [1.1.0] - 2026-02-25
### ✨ 기능 추가 (Added)
- **자막 뷰어 (초안)**: VTT 파서를 이용하여 유튜브 자막(한국어)을 뽑아와 재생 엠베드에 송출.
- **라이브 진행 바**: 노래가 재생되는 시간에 비례하여 동적으로 길이가 바뀌는 ASCII 기반의 재생 진행 아이콘 표시 지원 개시.

---

## [1.0.0] - 2026-02-20
### 🚀 초기 릴리즈 (Initial Release)
- 기본 유튜브 음악 재생(Play/Pause/Skip) 및 `yt-dlp`, FFmpeg를 활용한 봇 구동 환경 구축 완료.
- Dockerizing 세팅 및 메인 스켈레톤 프로젝트 연동 완료.
